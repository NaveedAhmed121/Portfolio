Rail Incident Analyst Agent

Python · Anthropic Claude API · pandas · Docker · PostgreSQL · pytest

An agentic AI pipeline that ingests simulated rail/transit incident reports, classifies each by severity and type using the Claude API, generates corrective-action recommendations, and stores structured results in PostgreSQL — all containerized with Docker.

  • Built a multi-turn agent loop with structured JSON output, automatic retry on parse failure, and fallback classification to ensure 100% incident coverage even on API errors
  • Used pandas for CSV ingestion, column validation, and deduplication of 20+ realistic Toronto transit incident records
  • Containerized the full stack with Docker and docker-compose; PostgreSQL backend stores results with indexed queries on severity and category
  • Achieved 7/7 unit tests passing (pytest) covering success path, fallback logic, JSON fence-stripping, and analytics aggregation
  • Domain scoped to rail transit — demonstrating applied AI in operational monitoring and incident management contexts
